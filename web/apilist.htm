evolve API List

The following API URLs return data from the python component of evolve in JSON format. Some require parameters to return data specific to that identifier while others provide more general data. If viewing manually, it will be easier to read with a JSON tool or plugin. All links will open in a new tab/window via 'target=_blank' attribute.

Read Only API with no parameters

The following are read only URLs that return data outright. Clicking on the links will result in data being display without any further actions.

Plugin List: {get}
/data/plugins
Return a list of available volatility plugins based on the currently used profile. This will return various additional pieces of data to include morphs, row counts, errors, etc.

Evolve Metadata: {get}
/data/meta
Return a small set of data with the volatility and evolve version, memdump path, and profile selected.

Volatility Profile List: {get}
/data/profilelist
Return a list of Volatility profiles available.

Volatility Command Options List: {get}
http://localhost:8080/data/optslist
Return a list of command line options. (also available with a parameter)

Read Only API with parameters

The follwing are read only URLs that return data with parameter input. At least one will be required with others being noted as optional.

Plugin Data: {get / post}
/data/view/[name]
/data/view/[name]/morph/[morph]
Return the data generated from Volatility for the named plugin. This call will not cause the plugin to run against the memdump; the data must already be in the database. The second URL returns the same data with the column values being potentially modified by the morph as named.
Parameters:

  • name {get}: Name of volatility plugin.
  • query {post} [optional]: SQL statement to perform against plugin data.
  • morph {get} [optional]: Name of morph to apply to plugin data.

    Process ID Profile: {get}
    /data/view/pidprofile/[pid]
    Return a result set across static list of tables that contain a process id (PID) or parent process id (PPID).
    Parameters:

  • pid {get}: Process identifier to search for.

    Volatility Command Options List - Plugin Specific: {get}
    http://localhost:8080/data/optslist/plugin/[name]
    Return a list of plugin specific command line options with default options removed.
    Parameters:

  • name {get}: Name of plugin.

    Writing API with parameters

    The follwing are URLs that do not return data and instead start a process or place data into the configuration or database.

    Set Volatility Profile: {get}
    /config/profile/[pname]
    Set the configuration of evolve to use the named Volatility profile for future plugin runs.
    Parameters:

  • pname {get}: Name of volatility profile.

    Run Volatility Plugin: {get}
    /run/plugin/[name]
    Start a Volatility plugin through Python Multiprocess against the memdump.
    Parameters:

  • name {get}: Name of volatility plugin to run.
